CASE STUDIES
INVESTIGATION — 001
DNS Beaconing Detection Investigation
T1071 · C2
THREAT SCENARIO
Suspicious periodic DNS activity suggested a low-and-slow beacon pattern that required validation against endpoint and network telemetry.
DETECTION METHOD
Relied on Sysmon telemetry and network logs. Recurring query behavior and process context used to confirm the signal was not routine background traffic.
CORRELATION PROCESS
Cross-referenced Sysmon telemetry with network logs to separate suspicious behavior from background noise. Validated timing intervals and process parentage chains.
OUTCOME
Validated command-and-control behavior and provided a clear ATT&CK-mapped evidence trail for SOC review.
T1071 — Application Layer Protocol
INVESTIGATION — 002
PowerShell Abuse & LOLBIN Detection
T1059.001 · EXEC
THREAT SCENARIO
Encoded PowerShell execution observed in process telemetry, potentially used for defense evasion and payload delivery via living-off-the-land techniques.
DETECTION METHOD
Sysmon Event ID 1 (process creation) and Event ID 3 (network connection) correlated to surface encoded execution patterns and suspicious parent-child process chains.
CORRELATION PROCESS
Mapped LOLBIN usage against known execution paths. Decoded payload strings and traced network callbacks to identify C2 infrastructure indicators.
OUTCOME
Confirmed malicious use of native Windows tools for execution. Detection rule tuned and mapped to ATT&CK framework.
